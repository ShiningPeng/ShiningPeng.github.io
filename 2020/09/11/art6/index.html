事情多

  • 第一天去上班把伞落在共享电动车上，无了，大学掉的第三把伞，这两天买的遮阳伞到了，洗完澡发现又黑了，害，一言难尽。

  • 还没有在学校办理实习申请，因为第一周可以退改选，所以直接没去上课去上班了，平时也没什么时间，第二周再找老师申请一下。

  • 学校的封闭管理好像不是很严，以前从寝室外围的洞里直接走出校，最近洞被封了，都是直接从校门出去，然后到公交站等车，一般都是提前十几二十分钟到公司。

  • 工作中体验到合作的感觉，我写前端页面，UI小哥问我有没有什么还需要再切的，和他沟通，感觉还挺好的，今天也问了带我的宇哥蛮多问题的，超过十分钟没有解决就立刻问，以面耽误进度，不过我今天进度还是有点慢，调结构和样式调了蛮久。所以打算之后自己写页面有些原则，写好结构调样式一直调不好，更换页面结构，换个角度思考解决方案。

记一些笔记

  • 给元素设置图片背景

    1. 将图片放在一个盒子中作为子元素，使用定位定好，但需要注意宽高的设置，否则容易出现拉伸图片或者超出盒子等问题，举个栗子
      <view class="top-wrapper">
    <view class="top-bgi">
      <img src="/static/pc-images/banner.png" alt />
    </view>
    <view class="top-content">
      ...
    </view>
</view>
      .top-wrapper {
width: 100%;
height: 7.73rem;
position: relative;
z-index: 1;
display: flex;
align-items: flex-end;
justify-content: center;
}
.top-bgi {
  height: 100%;
  position: absolute;
  top: 0;
  left: 0;
  z-index: -1;
}
.top-bgi img {
  width: 100vw;
  line-height: 0;
  height: 100%;
}
    • 主要是和宽高和定位有关的样式，其他的一些样式是用作别的功能的。
    • 定位用作父子元素，父元素设置position:relative;子元素设置position:absolute;然后通过top,left,right,bottom来控制子元素的位置
    1. 给这个元素设置background属性，并且设置几个相关的属性进行调整便可
    <view class="img-wrapper">
</view>
``` 

```css
.img-wrapper {
width: 4.53rem;
height: 2.09rem;
background: url(/static/pc-images/judge.png);
background-size: 100% 100%;
}

  • 写页面时尽量使用弹性布局，以及使用rem单位等从多个方面考虑兼容性，暂时我写的是pc端，可能这块考虑的不是很多，后面有机会接触移动端再总结。

加速
